## ✅ 문제

회사는 최근 청구서를 통해 Amazon EC2 비용이 증가했음을 확인했다.

청구 팀은 일부 EC2 인스턴스에서 **원치 않는 수직 확장(Vertical Scaling)** 이 발생한 것을 발견했다.

솔루션 아키텍트는 다음을 수행해야 한다.

- 최근 2개월간 EC2 비용을 비교하는 그래프 생성
- 수직 확장의 근본 원인에 대한 심층 분석 수행
- 최소한의 운영 오버헤드로 정보 생성

---

## 📌 요구사항 분석

이 문제는 단순한 "비용 확인" 문제가 아니다.  
핵심은 다음 세 가지다.

### 1️⃣ 최근 2개월간 EC2 비용 비교 그래프 생성

- 특정 기간(최근 2개월)
- EC2 비용만 필터링
- 인스턴스 유형 기준 비교 필요

---

### 2️⃣ 수직 확장의 근본 원인 심층 분석

#### 🔎 수직 확장(Vertical Scaling)

- 하나의 서버 성능을 향상시키는 방식
- CPU, RAM, 스토리지 등을 상향 조정
- 구현은 단순하지만 비용 증가 가능성 높음
- 인스턴스 타입이 변경되었는지 확인 필요

즉,
- m5.large → m5.4xlarge 처럼 업그레이드 되었는지
- 특정 인스턴스 타입 비용이 급증했는지
를 분석해야 한다.

---

### 3️⃣ 최소한의 운영 오버헤드

#### 🔎 운영 오버헤드란?

- 추가 설정 작업
- 별도의 인프라 구축
- 데이터 가공 및 파이프라인 구성
- 보고서 자동화 구축 작업

즉,
"복잡한 구성 없이 바로 분석 가능해야 한다"는 의미다.

---

##  요구사항과 관련된 AWS 서비스

### 🔹 AWS Cost Explorer
- AWS 비용과 사용량을 시각적으로 분석
- 서비스별 비용 비교 가능
- 인스턴스 유형별 필터링 가능
- 특정 기간 설정 가능
- 별도 인프라 구성 불필요

---

### 🔹 AWS Budgets
- 비용 목표 설정
- 초과 시 알림
- 주로 예측/모니터링 목적

---

### 🔹 AWS Billing and Cost Management 대시보드
- 전체 비용 개요 제공
- 고급 필터링 기능은 제한적

---

### 🔹 AWS Cost and Usage Report (CUR)
- 매우 세부적인 비용 데이터 제공
- S3에 저장 후 Athena/QuickSight 분석 가능
- 설정 복잡
- 운영 오버헤드 증가 가능

---

### 🔹 Amazon QuickSight
- BI 시각화 도구
- 다양한 데이터 소스를 기반으로 대시보드 구성
- 추가 구성 필요

---

## 🎯 선택지 분석

### ❌ 선택지 1  
**AWS Budgets 사용**

- 예산 초과 알림 중심 기능
- 세부적인 인스턴스 유형별 비용 분석 기능 제한
- 고급 필터링 기능 부족

→ 고수준 모니터링 도구이지, 심층 분석 도구가 아니다.

---

### ✅ 선택지 2  
**Cost Explorer의 세분화된 필터링 기능 사용**

- 최근 2개월 기간 설정 가능
- EC2 서비스만 필터링 가능
- 인스턴스 유형 기준 그룹화 가능
- 비용 추이 그래프 생성 가능
- 별도 인프라 구성 필요 없음
- 운영 오버헤드 최소

→ 요구사항(그래프 + 심층 분석 + 최소 오버헤드) 모두 충족

---

### ❌ 선택지 3  
**Billing and Cost Management 기본 대시보드 사용**

- 전체 비용 개요 중심
- 세부 필터링/분석 기능 제한
- 인스턴스 유형 기준 심층 분석에 적합하지 않음

---

### ❌ 선택지 4  
**Cost and Usage Report + S3 + QuickSight**

- 가장 세부적인 데이터 제공
- 대화형 그래프 구성 가능
- 하지만 설정 복잡
- 추가 서비스 구성 필요
- 운영 오버헤드 증가

→ 문제의 "최소한의 운영 오버헤드" 조건에 맞지 않음
