## ✅ 문제

회사는 NFS를 사용하여 온프레미스 네트워크 연결 스토리지에 대용량 비디오 파일을 저장하고 있다.

- 각 파일 크기: 1MB ~ 500GB
- 총 스토리지 용량: 70TB (더 이상 증가하지 않음)

회사는 이 비디오 파일을 Amazon S3로 마이그레이션하려고 한다.

### 요구 조건
- 최소한의 네트워크 대역폭 사용
- 가능한 한 빠르게 마이그레이션 수행

---

## 📌 요구사항 분석

이 문제의 핵심 요구사항은 다음과 같다.

### 1️⃣ 비디오 파일을 Amazon S3로 마이그레이션

- 파일 크기가 매우 다양함 (1MB ~ 500GB)
- 총 데이터 용량이 70TB로 대규모
- 네트워크 대역폭 사용을 최소화해야 함
- 빠른 마이그레이션 필요

---

## 🛠 요구사항과 관련된 AWS 서비스

### 🔹 AWS DataSync
- 온프레미스 스토리지를 Amazon S3로 전송하는 관리형 서비스
- 네트워크 최적화 기능 제공
- NFS와 통합 가능
- 네트워크를 통해 직접 S3로 데이터 전송

---

### 🔹 AWS Snowball
- 대용량 데이터를 AWS로 전송하기 위한 물리적 디바이스 솔루션
- 인터넷 대역폭이 제한적인 경우 유용
- 네트워크 사용을 최소화 가능
- 물리적으로 데이터를 AWS로 전송

---

### 🔹 AWS Storage Gateway (S3 File Gateway)
- 온프레미스 데이터를 S3로 전송하는 하이브리드 스토리지 솔루션
- 기존 NFS 파일 공유를 S3로 마이그레이션 가능
- 네트워크를 통해 데이터 전송

---

### 🔹 AWS Transfer Family
- SFTP, FTPS, FTP를 통해 S3로 파일 전송
- 기존 파일 전송 클라이언트 사용 가능
- 네트워크 기반 전송 방식

---

### 🔹 AWS Direct Connect
- 온프레미스와 AWS 간 전용 네트워크 연결 제공
- 공용 인터넷 우회 가능
- 안정적이고 빠른 전송 지원
- 하지만 여전히 네트워크 기반 전송

---

## 🎯 선택지 분석

### ❌ 선택지 1
**S3 + IAM + AWS CLI로 직접 복사**

- 네트워크를 통해 직접 전송
- 70TB 대용량 전송 시 많은 대역폭 사용
- 전송 시간 오래 걸릴 가능성 있음

→ 최소 네트워크 사용 조건을 충족하지 못함

---

### ✅ 선택지 2
**AWS Snowball Edge 사용**

- 물리적 디바이스로 데이터 전송
- 네트워크 대역폭 최소 사용
- 70TB 대용량 데이터에 적합
- 빠른 전송 가능
- 고속 네트워크 인터페이스 제공

→ 요구사항 충족

---

### ❌ 선택지 3
**S3 File Gateway (공용 엔드포인트)**

- 네트워크 기반 전송
- 70TB 전송 시 시간 오래 걸릴 수 있음
- 네트워크 대역폭 사용 발생

→ 최소 대역폭 사용 조건 미충족

---

### ❌ 선택지 4
**Direct Connect + S3 File Gateway**

- 전용 회선 사용으로 안정성 향상
- 하지만 여전히 네트워크 기반 전송
- 70TB 전송 시 시간 소요 큼

→ 최소 대역폭 사용 조건 미충족
