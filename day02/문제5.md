## ✅ 문제

회사에는 1,000개의 Amazon EC2 Linux 인스턴스에서 실행되는 프로덕션 워크로드가 있다.

이 워크로드는 타사 소프트웨어로 구동된다.

회사는 중요한 보안 취약성을 수정하기 위해 가능한 한 빠르게 모든 EC2 인스턴스에 패치를 적용해야 한다.

솔루션 설계자는 이러한 요구 사항을 충족해야 한다.

---

## 📌 선택지

1. 모든 EC2 인스턴스에 패치를 적용할 AWS Lambda 함수를 생성한다.  
2. 모든 EC2 인스턴스에 패치를 적용하도록 AWS Systems Manager Patch Manager를 구성한다.  
3. 모든 EC2 인스턴스에 패치를 적용하도록 AWS Systems Manager 유지 관리 기간을 예약한다.  
4. AWS Systems Manager Run Command를 사용하여 패치를 모든 EC2 인스턴스에 적용하는 사용자 지정 명령을 실행한다.  

---

## 📌 요구사항 분석

이 문제에는 1가지 핵심 요구사항이 있다.  
하지만 세부적으로 살펴보면 다음과 같은 조건이 포함되어 있다.

### 1️⃣ 소프트웨어 패치

- 1,000개의 Amazon EC2 Linux 인스턴스에서 실행되는 프로덕션 워크로드
- 워크로드는 타사 소프트웨어로 구동됨
- 중요한 보안 취약성 발견
- 가능한 한 빠르게 모든 EC2 인스턴스에 패치 적용 필요

### 🔎 종합 정리

회사에서 사용 중인 타사 소프트웨어에 심각한 보안 문제가 발견되었고,  
이를 해결하기 위해 **1,000개의 모든 서버에 최대한 빠르게 업데이트를 적용해야 하는 상황**이다.

---

## 🛠 요구사항과 관련된 AWS 서비스

### 🔹 AWS Systems Manager
대규모 인스턴스 관리 및 자동화를 지원하는 서비스다.  
인스턴스 상태를 모니터링하고 제어할 수 있으며, 여러 EC2 인스턴스를 중앙에서 관리할 수 있다.

---

### 🔹 AWS Systems Manager Automation
타사 소프트웨어를 포함한 다양한 워크로드를 자동화된 방식으로 관리할 수 있다.  
패치 작업을 자동으로 수행하는 데 유용하다.

---

### 🔹 AWS Systems Manager Patch Manager
EC2 인스턴스에 자동으로 패치를 적용할 수 있는 기능이다.  
보안 패치를 포함한 다양한 패치 작업을 신속하게 수행할 수 있다.

---

### 🔹 AWS Systems Manager Run Command
여러 인스턴스에 대해 원격으로 명령을 실행할 수 있는 서비스다.  
긴급한 패치 작업을 즉시 수행할 수 있다.

---

## 🎯 선택지 분석

### ❌ 선택지 1  
**AWS Lambda 함수를 생성하여 패치 적용**

- Lambda는 서버리스 컴퓨팅 서비스
- 일반적으로 짧은 실행 시간에 적합
- 1,000개의 EC2 인스턴스 패치에는 비효율적
- 운영 오버헤드 증가 가능

→ 대규모 패치 작업에는 적합하지 않다.

---

### ❌ 선택지 2  
**AWS Systems Manager Patch Manager 구성**

- 패치 작업 자동화 가능
- 보안 패치 적용을 지원

→ 자동화에는 적합하지만,  
즉각적인 긴급 패치 실행에는 Run Command가 더 적합하다.

---

### ❌ 선택지 3  
**AWS Systems Manager 유지 관리 기간 예약**

- 패치 및 유지 관리 작업을 예약 가능
- 운영 오버헤드 감소

→ 예약 기반 방식이므로  
긴급 보안 패치의 즉각적인 적용에는 적합하지 않다.

---

### ✅ 선택지 4  
**AWS Systems Manager Run Command 사용**

- 여러 인스턴스에 동시에 명령 실행 가능
- 긴급 패치 작업을 즉시 수행 가능
- 운영 오버헤드 최소화

→ 가능한 한 빠르게 모든 인스턴스에 패치를 적용해야 하는 요구사항을 충족한다.
